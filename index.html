Règle centrale : Claude ne doit jamais envoyer d’e-mail automatiquement.

1. 🧱 Créer le projet Claude Cowork

Nom proposé : Agentic pour le Cabinet JCA

Prompt — Instructions projet

Ce projet sert à prototyper deux cas d’usage :
1. gestion des mails Outlook non répondus ;
2. gestion des verbatims clients.

Règles générales :
- ne jamais envoyer d’e-mail automatiquement ;
- ne jamais promettre une action non validée par un humain ;
- travailler par étapes simples ;
- tenir un fichier Excel comme base de suivi ;
- ne pas créer de fichier cloud parallèle ;
- utiliser des colonnes et valeurs en anglais dans le fichier Excel ;
- traiter uniquement les nouveaux éléments ou les éléments modifiés ;
- limiter les volumes traités à chaque exécution ;
- produire un rapport court à chaque run ;
- utiliser la skill xlsx-jca pour créer, lire et modifier le fichier Excel ;
- utiliser la skill pdf-jca pour générer les rapports PDF ;
- ne créer la Skill métier qu’après plusieurs étapes testées dans ce fil ;
- créer la route finale seulement après validation du workflow et de la Skill métier.

2. 🔌 Vérifier Outlook, le projet et les skills

Objectif : vérifier Microsoft 365, le répertoire projet, xlsx-jca et pdf-jca avant toute action.

Prompt — Vérification des accès

Vérifie que tu as accès aux éléments nécessaires pour ce projet :

1. Connecteur Microsoft 365 :
- peux-tu lire les mails Outlook ?
- peux-tu retrouver les fils de discussion Outlook ?
- peux-tu identifier les mails entrants et sortants ?

2. Répertoire du projet Claude Cowork :
- peux-tu créer un fichier dans le répertoire du projet ?
- peux-tu retrouver ce fichier ensuite ?
- peux-tu le modifier lors d’un prochain run ?

3. Skill tableur :
- la skill xlsx-jca est-elle disponible ?
- peux-tu l’utiliser pour créer, lire et modifier un fichier .xlsx dans le projet ?

4. Skill PDF :
- la skill pdf-jca est-elle disponible ?
- peux-tu l’utiliser pour créer un rapport PDF à partir d’un fichier Excel ou d’un résumé de run ?

Ne lance pas encore le traitement des mails.
Ne crée pas encore de brouillon.
Ne crée pas encore de fichier PDF.
Ne modifie rien sans confirmation.

3. 📊 Créer le fichier Excel

Fichier cible : Base mails Outlook.xlsx

Colonnes minimales :
email_id, thread_id, received_at, sender, subject, snippet, has_attachment, status, priority, signal, suggested_reply, human_check
Valeurs autorisées :
status = to_analyze / unanswered / draft_proposed / done / ignored
priority = critical / high / medium / low
signal = urgency / client_frustration / budget_issue / operational_blocker / sales_request / follow_up / none
has_attachment = yes / no
human_check = required / not_required

Prompt — Création du fichier Excel

Utilise la skill xlsx-jca pour créer un fichier Excel intitulé “Base mails Outlook.xlsx” dans le répertoire du projet Claude Cowork.

Le fichier doit contenir un tableau structuré avec les colonnes suivantes, en anglais :
- email_id
- thread_id
- received_at
- sender
- subject
- snippet
- has_attachment
- status
- priority
- signal
- suggested_reply
- human_check

Règles :
- ne pas encore importer les mails ;
- créer seulement la structure du fichier ;
- utiliser uniquement ces colonnes pour le prototype ;
- ne pas créer de fichier cloud parallèle ;
- vérifier que le fichier est bien sauvegardé dans le répertoire du projet ;
- indiquer le chemin du fichier créé.

4. 📥 Importer Outlook dans Excel

Objectif : lire Outlook via Microsoft 365 et écrire les mails utiles dans le fichier Excel.

Prompt — Import Outlook vers Excel

Utilise Outlook via le connecteur Microsoft 365 pour lire les mails présents dans la boîte de réception.

Utilise la skill xlsx-jca pour remplir le fichier Excel “Base mails Outlook.xlsx” situé dans le répertoire du projet Claude Cowork.

Pour chaque mail utile, ajoute une ligne avec :
- email_id si disponible ;
- thread_id si disponible ;
- received_at ;
- sender ;
- subject ;
- snippet court ;
- has_attachment = yes/no ;
- status = to_analyze.

Pour les colonnes non encore analysées, laisse vide :
- priority ;
- signal ;
- suggested_reply ;
- human_check.

Ne rédige aucune réponse.
Ne classe pas encore les mails.
Ne crée aucun brouillon.
Ignore les newsletters, no-reply et notifications automatiques si tu les identifies clairement.
Ne crée pas de fichier cloud parallèle.

À la fin, indique :
- chemin du fichier Excel ;
- nombre de mails ajoutés ;
- nombre de mails ignorés ;
- limites rencontrées ;
- erreurs éventuelles.

5. 🔎 Détecter les mails non répondus

Colonne mise à jour : status.

Prompt — Détection des mails non répondus

À partir du fichier Excel “Base mails Outlook.xlsx”, identifie les fils de discussion où le dernier message vient d’un client ou contact externe, et où aucune réponse sortante n’a été envoyée après ce dernier message.

Utilise la skill xlsx-jca pour mettre à jour uniquement la colonne :
- status

Valeurs autorisées :
- unanswered
- ignored
- to_analyze

Ne crée pas de nouvelles colonnes.
Ne génère pas de brouillon.
Ne rédige pas encore de réponse.
Ne modifie pas les mails dans Outlook.
Ne crée pas de fichier cloud parallèle.

6. 🚦 Prioriser les mails

Colonnes mises à jour : priority et signal.

Prompt — Priorisation

Analyse uniquement les lignes où status = unanswered dans le fichier Excel “Base mails Outlook.xlsx”.

Classe chaque mail avec la colonne :
- priority

Valeurs autorisées :
- critical
- high
- medium
- low

Détecte le signal principal avec la colonne :
- signal

Valeurs autorisées :
- urgency
- client_frustration
- budget_issue
- operational_blocker
- sales_request
- follow_up
- none

Utilise la skill xlsx-jca pour mettre à jour uniquement :
- priority
- signal

Ne prépare pas encore de brouillon.
Limite l’analyse aux 30 mails les plus récents.
Ne crée pas de fichier cloud parallèle.

7. ✍️ Proposer des réponses à valider

Uniquement pour status = unanswered et priority = critical ou high.

Prompt — Propositions de réponse

Pour les lignes où status = unanswered et priority = critical ou high, prépare une proposition de réponse courte, professionnelle et factuelle.

Maximum 10 propositions.

Ne jamais envoyer d’e-mail automatiquement.

Utilise la skill xlsx-jca pour mettre à jour uniquement :
- suggested_reply
- human_check
- status

Règles :
- suggested_reply doit être court ;
- human_check = required ;
- status = draft_proposed ;
- ne promets jamais une action non confirmée ;
- signale dans la réponse les points à vérifier si nécessaire ;
- ne crée pas de nouvelles colonnes ;
- ne crée pas de fichier cloud parallèle.

Si possible, crée un brouillon Outlook, mais ne l’envoie jamais.

8. 📄 Générer le rapport PDF

Objectif : générer un PDF court, lisible et auditable avec pdf-jca.

Prompt — Rapport PDF

À partir du fichier Excel “Base mails Outlook.xlsx”, utilise la skill pdf-jca pour générer un rapport PDF court du run.

Le rapport doit contenir :
- date du run ;
- nombre de mails scannés ;
- nombre de mails ajoutés ;
- nombre de mails ignorés ;
- nombre de mails avec status = unanswered ;
- nombre de priorités critical / high / medium / low ;
- principaux signaux détectés ;
- 3 exemples anonymisés de cas critical ou high ;
- propositions de réponse à valider ;
- limites rencontrées ;
- actions humaines recommandées.

Règles :
- ne jamais inclure de données sensibles inutiles ;
- ne jamais inclure le contenu complet d’un e-mail ;
- anonymiser les exemples si possible ;
- ne jamais envoyer ce rapport automatiquement ;
- sauvegarder le PDF dans le répertoire du projet Claude Cowork ;
- indiquer le chemin du PDF généré.

12. 🛣️ Créer la route finale

La route rejoue le workflow validé.

Prompt — Création de la route finale

Crée une route opérationnelle pour le projet “Agentic pour le Cabinet JCA”.

Cette route doit utiliser la Skill métier :
managing-outlook-inbox-emails

Objectif de la route :
mettre à jour le fichier Excel “Base mails Outlook.xlsx” dans le répertoire du projet Claude Cowork à partir des nouveaux mails Outlook ou des fils modifiés.

La route doit exécuter les étapes suivantes :
1. vérifier l’accès Outlook via Microsoft 365 ;
2. vérifier l’accès au fichier “Base mails Outlook.xlsx” ;
3. utiliser xlsx-jca pour lire et mettre à jour le fichier Excel ;
4. scanner uniquement les nouveaux mails ou fils modifiés ;
5. ignorer newsletters, no-reply, notifications automatiques et alertes système ;
6. ajouter les nouveaux mails utiles avec status = to_analyze ;
7. détecter les mails probablement non répondus avec status = unanswered ;
8. prioriser les cas avec priority = critical / high / medium / low ;
9. renseigner le signal principal avec signal ;
10. proposer une réponse uniquement pour les cas critical ou high ;
11. mettre human_check = required pour toute proposition ;
12. ne jamais envoyer d’e-mail ;
13. ne créer aucun fichier cloud parallèle ;
14. générer un rapport court dans Claude ;
15. générer un rapport PDF avec pdf-jca dans le répertoire du projet.

Limites :
- maximum 50 mails scannés ;
- maximum 30 mails analysés ;
- maximum 10 propositions de réponse ;
- ne pas retraiter tout l’historique ;
- demander validation humaine en cas d’ambiguïté.

À la fin de chaque exécution, la route doit indiquer :
- nombre de mails scannés ;
- nombre de mails ajoutés ;
- nombre de mails ignorés ;
- nombre de mails avec status = unanswered ;
- nombre de cas critical / high ;
- nombre de propositions générées ;
- chemin du fichier Excel mis à jour ;
- chemin du rapport PDF généré ;
- actions humaines à valider.

Prompt — Test de la route finale

Teste la route managing-outlook-inbox-emails sur un périmètre réduit.

Contraintes de test :
- traiter uniquement les 20 derniers mails Outlook ;
- ne créer aucun brouillon ;
- ne jamais envoyer d’e-mail ;
- ne créer aucun fichier cloud parallèle ;
- utiliser xlsx-jca pour écrire dans “Base mails Outlook.xlsx” ;
- utiliser pdf-jca pour générer le rapport PDF uniquement si le test est concluant.

Objectif du test :
vérifier que la route respecte :
- les colonnes autorisées ;
- les valeurs autorisées ;
- les limites de volume ;
- les exclusions ;
- le connecteur Microsoft 365 ;
- xlsx-jca ;
- pdf-jca ;
- l’interdiction d’envoi automatique ;
- l’interdiction de fichier cloud parallèle.
